(상위로직을 만들어서 위에 파일을 삽입, 삭제 기능 구현)

3차 업데이트(ec2_to_s3_1th_final_2nd.py)
- 크게 수정사항은 없지만, 버킷 이후 경로를 번호보다는 많아질 수 있지만, 인풋으로해서 직접 입력하는걸로 바꾸기

4차 업데이트((ec2_to_s3_1th_final_3rd.py)
- 3차 업데이트 이후 코드를 경량화하는데 집중
- 업로드하는 폴더 내의 경로가 s3 버킷내 db 아래 경로로 지정이 되어야되서
  해당 부분으로 만드는 로직 설정.
- 업로드하는 ec2 내의 경로와 경로 내의 모든 파일들을 파악하는 알고리즘 도입

5차 업데이트(ec2_to_s3_1th_final_3rd_re.py)
- 업로드하는 파일 경로를 s3 에 업로드할 때 경로에 넣어서 자동으로 파일 생성시 폴더로 되게끔 만듬
- 최대한 필요없는 로직들 정리
- 4차업데이트와 마찬가지로 업로드하면 중복된 파일이 존재할 경우 삭제하고 다시 덮어쓰고 새로운 파일일 경우 그냥 새로설치되게함
- 그리고 업데이트된 파일, 추가된 파일이 리스트에 잡혀서 어떤것이 업데이트 되엇고, 새로 추가되었는지 파악 가능
